# Plant

## 專案架構

```
Plant/
├── main.py                 # 程式入口點
├── flow.py                 # 流程控制器
├── model.py                # LLM 模型整合層（OpenAI, Anthropic, Gemini, Ollama）
├── store.py                # I/O 層：JSON/Markdown 讀寫
├── utils.py                # Logger 和 MoM Manager
│
├── agents/                 # AI Agents
│   ├── __init__.py
│   ├── user.py            # UserAgent: 扮演利害關係人
│   ├── analyst.py         # AnalystAgent: 需求分析與衝突識別
│   ├── expert.py          # ExpertAgent: 領域專家建議（RAG）
│   ├── mediator.py        # MediatorAgent: 調解與決策
│   ├── modeler.py         # ModelerAgent: 系統建模（PlantUML, AST）
│   └── documentor.py      # DocumentorAgent: 文件生成
│
├── config/                 # 配置文件
│   ├── config.json        # 系統配置
│   └── spec.json          # Draft 和 IEEE 29148 模板
│
├── doc/                    # 外部參考文件（RAG 知識來源）
│   ├── .gitkeep           # 資料夾說明
│   ├── *.txt              # 法規、標準文件
│   ├── *.md               # 技術文件
│   └── *.json             # 結構化參考資料
│
├── artifact/               # 中間產物
│   ├── artifact.json      # 核心數據存儲
│   ├── draft.json         # 需求草稿
│   ├── srs.json           # 正式 SRS
│   ├── uml.json           # UML 模型
│   └── mom.json           # 會議記錄（JSON）
│
├── output/                 # 輸出文件
│   ├── report.md          # 衝突報告
│   ├── uml.md             # UML 圖表
│   ├── dr.md              # 設計理由（Design Rationale）
│   ├── srs.md             # 正式 SRS 文件
│   └── mom.md             # 會議記錄（Markdown）
│
└── log/                    # 日誌文件
    └── system.log
```

## 🔄 執行流程

### 單輪完整流程

```
人類輸入粗略想法 (rough_idea)
    ↓
┌─────────────────────────────────────────────────────────────┐
│ Stage 1: Mediator                                           │
│ ├─ 1.1 產生系統概述 (system_description)                   │
│ ├─ 1.2 提出 5-8 位利害關係人建議                            │
│ └─ 1.3 人類選擇至多 5 位利害關係人                          │
└─────────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────────┐
│ Stage 2: User Agent                                         │
│ └─ 扮演利害關係人提出需求（基於系統概述）                   │
│    - 操作流程                                                │
│    - 期待功能                                                │
│    - 實際使用情境                                            │
└─────────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────────┐
│ Stage 3: Analyst Agent                                      │
│ ├─ 3.1 配對分析利害關係人需求                               │
│ ├─ 3.2 識別衝突並產生衝突報告                               │
│ └─ 3.3 輸出 report.md                                       │
└─────────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────────┐
│ Stage 4: Expert Agent（可選）                               │
│ ├─ 載入外部文件（RAG）- doc/ 資料夾                         │
│ └─ 提供專業建議                                              │
│    - 法規與合規性                                            │
│    - 資料安全                                                │
│    - 系統效能                                                │
│    - 可維護性與可測試性                                      │
└─────────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────────┐
│ Stage 5: Mediator                                           │
│ ├─ 5.1 根據衝突產生決策選項                                 │
│ └─ 5.2 人類裁決（可跳過）                                   │
└─────────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────────┐
│ Stage 6: Mediator                                           │
│ └─ 整合 artifact.json 產生 draft.json                       │
│    （System Overview = system_description）                 │
└─────────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────────┐
│ Stage 7: Modeler Agent（可選）                              │
│ └─ 產生系統模型                                              │
│    - PlantUML 圖表                                           │
│    - AST（抽象語法樹）                                       │
│    - 輸出 uml.md                                             │
└─────────────────────────────────────────────────────────────┘
    ↓
    保存 MoM（接續累積，不覆蓋）
    ↓
┌─────────────────────────────────────────────────────────────┐
│ 最終階段: Documentor Agent（所有輪次結束後）                │
│ ├─ 從 MoM 產生設計理由 (dr.md)                              │
│ │  - 決策理由                                                │
│ │  - 方案取捨過程                                            │
│ │  - 替代方案                                                │
│ │  - 依據與參考                                              │
│ ├─ 將 draft.json 轉換為 IEEE 29148 格式                     │
│ ├─ 輸出 srs.json 和 srs.md                                  │
│ └─ 輸出 mom.json 和 mom.md                                  │
└─────────────────────────────────────────────────────────────┘
```

### 多輪精煉

第 2+ 輪：
- **User Agent**: 精煉現有利害關係人需求（基於 previous_draft）
- **Expert Agent**: 精煉專家建議（支援外部文件 RAG）
- **Modeler Agent**: 調整現有模型（增刪改）
